Регламентные работы

Данный раздел предназначен для описания стандартных процедур и задач, которые необходимо выполнять в рамках проекта для обеспечения его стабильной работы и достижения поставленных целей.

Основные сроки проведения регламентных работ

Название регламентной работы Периодичность
Резервное копирование базы данных и бэкап кода 1 раз в сутки
Очистка кэша и старых логов 1 раз в неделю
Проверка обновлений зависимостей (Node.js + React) 1 раз в месяц
Оптимизация MySQL и мониторинг нагрузки 1 раз в месяц

Резервное копирование базы данных и бэкап кода

  • Бэкап базы данных MySQL
    • Откройте терминал
      • На Linux/Mac:
        • Нажмите Ctrl+Alt+T или найдите "Terminal" в меню приложений
      • На Windows:
        • Откройте "Пуск" → введите "Ubuntu" → нажмите Enter
    • Выполните команду
      • mysqldump -u admin -p12345 my_database > /backups/db_$(date +\%F).sql
      • Где:
        • admin - ваш логин от MySQL
        • 12345 - пароль (после -p пишется без пробела!)
        • my_database - название вашей базы данных
    • Проверьте, что файл создался
      • ls -lh /backups/
      • Должен появиться файл вида db_2024-01-15.sql (с текущей датой)
  • Бэкап кода приложения
    • В том же терминале выполните
      • sudo tar -czvf /backups/app_$(date +\%F).tar.gz /var/www/app
      • Введите пароль администратора при запросе
    • Проверьте архив
      • ls -lh /backups/
      • Должен появиться файл app_2024-01-15.tar.gz
  • Проверка бэкапов
    • Посмотрите список файлов
      • cd /backups/
      ls -lh
    • Что проверять
      • Размер файлов (не должен быть 0)
      • Дату создания (сегодняшняя)
      • Права доступа (-rw-r--r--)
  • Отправка в облако
    • Откройте браузер → зайдите на disk.yandex.ru
    • Нажмите "Загрузить" (синяя кнопка вверху слева)
    • Перетащите файлы из папки /backups/ в окно браузера

Очистка кэша и старых логов

  • Очистка кэша React
    • Откройте терминал в папке проекта
      • Windows:
        • Нажмите Win + R, введите cmd, откройте командную строку
        • Перейдите в папку проекта: cd C:\projects\ваш-проект\
      • Linux/macOS:
        • Откройте терминал (Ctrl+Alt+T или через поиск)
        • Перейдите в папку проекта: cd ~/projects/ваш-проект/
    • Запустите сборку проекта
      • npm run build
      • Что произойдет?
        • React пересоберет проект, удалив старый кэш
        • В папке появится директория /build с актуальными файлами
      • Если ошибка: npm install # Установит недостающие зависимости
        npm run build # Повторите сборку
  • Очистка кэша Node.js (Redis)
    • Откройте Redis CLI
      • Запустите терминал
      • Введите: redis-cli
    • Очистите весь кэш
      • FLUSHALL
      • Должен появиться ответ OK
    • Выйдите из Redis
      • exit
    • Проверка:
      • redis-cli KEYS "*" # Должен вернуть пустой список
  • Удаление старых логов сервера
    • Очистка логов Nginx
      • Откройте терминал с правами администратора: sudo su # Для Linux/macOS
      • Для Windows: Win → В поиске напишите "Терминал" → Нажмите на него правой кнопкой мыши → "Запуск от имени администратора"
      • Удалите логи старше 7 дней: sudo find /var/log/nginx -name "*.log" -mtime +7 -delete
      • Что делает команда?
        • Ищет файлы .log в /var/log/nginx/
        • Удаляет те, что созданы более 7 дней назад
    • Очистка системных логов (journalctl)
      • sudo journalctl --vacuum-time=7d
      • Что произойдет?
        • Система удалит логи, которым больше 7 дней
        • Появится сообщение "Vacuuming done, freed X GB"

Проверка обновлений зависимостей (Node.js + React)

  • Проверка обновлений зависимостей (Node.js + React)
    • Обновить npm-пакеты
      • Проверьте устаревшие пакеты: npm outdated
      • Обновите все пакеты: npm update
      • Что произойдет:
        • Будут обновлены все пакеты, для которых доступны новые версии
        • Файл package-lock.json будет автоматически обновлен
    • Проверить уязвимости
      • Запустите проверку безопасности: npm audit
      • Исправьте уязвимости: npm audit fix
      • Для критических исправлений: npm audit fix --force
      • Что проверять:
        • Количество уязвимостей в отчете
        • Типы уязвимостей (low, moderate, high, critical)
    • Обновить React
      • Установите последние версии: npm install react@latest react-dom@latest
      • Проверьте версии в package.json: grep '"react"' package.json
      • Важно:
        • Перед обновлением сделайте бэкап проекта
        • Проверьте changelog React на breaking changes
    • Пересобрать проект
      • Выполните сборку: npm run build
      • Проверьте результат:
        • Должна создаться папка build/
        • В консоли не должно быть ошибок
      • Если есть ошибки:
        • rm -rf node_modules/ && npm install && npm run build
  • Дополнительные рекомендации
    • Частота проверки обновлений:
      • Для production: 1 раз в месяц
      • Для активной разработки: 1 раз в неделю
    • Перед обновлением:
      • Создайте git-ветку для тестирования
      • Проверьте работу на тестовом стенде
    • Полезные команды:
      • Просмотр истории версий пакета: npm view react versions
      • Обновление конкретного пакета: npm install package-name@latest

Оптимизация MySQL и мониторинг нагрузки

  • Оптимизация MySQL
    • Проверить медленные запросы
      • Включите лог медленных запросов:
        • SET GLOBAL slow_query_log = 'ON';
        SET GLOBAL long_query_time = 1;
      • Просмотр логов:
        • sudo tail -f /var/log/mysql/mysql-slow.log
      • Что анализировать:
        • Запросы, выполняющиеся дольше 1 секунды
        • Частые повторяющиеся запросы
        • Запросы без использования индексов
    • Оптимизировать таблицы
      • Основная команда оптимизации:
        • OPTIMIZE TABLE tasks, projects;
      • Анализ использования таблиц:
        • ANALYZE TABLE problematic_table;
      • Рекомендации:
        • Выполняйте ночью для больших таблиц
        • Для InnoDB используйте pt-online-schema-change
        • Перед оптимизацией сделайте бэкап
    • Проверить индексы
      • Просмотр индексов таблицы:
        • SHOW INDEX FROM table_name;
      • Добавление индекса:
        • ALTER TABLE table_name ADD INDEX index_name (column_name);
      • Проверка эффективности:
        • Используйте EXPLAIN для проблемных запросов
        • Следите за Cardinality в SHOW INDEX
        • Удаляйте неиспользуемые индексы
    • Мониторинг нагрузки
      • Текущие процессы:
        • SHOW PROCESSLIST;
      • Статус сервера:
        • SHOW STATUS;
      • Нагрузка в реальном времени:
        • Для Linux:
          • sudo apt install mytop
          mytop -u root -p
        • Для Windows:
          • MySQL Workbench → Performance Dashboard
  • Оптимальные настройки конфигурации
    • Основные параметры my.cnf/my.ini
      • Буферный пул InnoDB:
        • innodb_buffer_pool_size = 4G # 70-80% от RAM
      • Логирование:
        • slow_query_log = 1
        long_query_time = 1
      • Соединения:
        • max_connections = 200
        wait_timeout = 300
  • Полезные инструменты
    • Для анализа запросов:
      • pt-query-digest - анализ логов
      • EXPLAIN - план выполнения
    • Для мониторинга:
      • Prometheus + Grafana
      • Percona Monitoring
    • Для оптимизации:
      • pt-index-usage - неиспользуемые индексы
      • pt-online-schema-change - безопасные изменения

Описание системы управления проектами

Название системы: DevFlow

Назначение: DevFlow — это веб-приложение для управления IT-проектами, предназначенное для команд разработчиков, менеджеров и заказчиков. Система помогает эффективно планировать, контролировать и анализировать выполнение задач на всех этапах жизненного цикла проекта.

Ключевые возможности

  • Управление задачами
    • Создание, назначение и приоритезация задач
    • Добавление статусов проектам ("Готово", "В разработке", "На паузе")
  • Отслеживание прогресса
    • Календарное планирование
    • Визуализация прогресса через графики
    • Генерация отчетов о выполнении

Технологии

  • Frontend: React.js + TypeScript
  • Backend: Node.js (Nest.js)
  • База данных: MySQL
  • Хостинг: AWS/GitHub Pages

Для кого это?

  • Разработчики: Удобное отслеживание своих задач
  • Менеджеры: Контроль сроков и ресурсов
  • Заказчики: Прозрачность процесса через гостевой доступ
